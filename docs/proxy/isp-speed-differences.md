# 电信、联通、移动翻墙速度差距大的真相

> 同样的机场套餐，买来的是同一批节点。为什么群友能跑满 500M，轮到你却卡成 PPT？答案往往不在机场，而在于你家的**宽带运营商**。

## 一、国内"御三家"的国际出口现状

中国三大宽带运营商（电信、联通、移动）在建设国际互联网出口时，因历史原因和战略定位不同，通往海外的"高速公路"长得完全不一样。

### 1. 中国电信：底子最厚，但也最拥挤

电信是国内出口带宽最宽的"老大哥"，国际出口线路最多。然而，用电信的人实在太多了。

> **类比**：这就像一条双向 16 车道的超级高速公路——路面宽阔，但到了晚高峰挤满了重型卡车，照样堵得一塌糊涂。

没有"中转"或"专线"加持时，电信用户在晚高峰直连海外节点，大概率会遭遇严重卡顿。

### 2. 中国联通：科学上网的"天选之子"

联通在国内用户量不及电信，但国际出口带宽相对充裕。对于科学上网而言，联通堪称**"神仙宽带"**。

车少路宽，联通用户即使使用普通的直连节点，在晚高峰的测速表现也往往远优于电信和移动。很多老手在租房时，甚至会**为了丝滑翻墙而特意拉一条联通宽带**。

### 3. 中国移动：国内无敌，出海拉胯

移动基站与光纤铺设在国内无可匹敌，"办手机卡送宽带"策略使其用户量极其庞大。但它的致命弱点在于：

- **国际出口带宽极小**
- **路由路径奇葩**：发往日本的数据，非要先绕道香港再转一圈
- 在科学上网圈被称为 **"墙中墙"**，直连海外体验极差

| 运营商 | 国际出口带宽 | 晚高峰直连表现 | 科学上网友好度 |
| :--- | :--- | :--- | :--- |
| **中国电信** | 最宽 | 中等（人多易拥挤） | ⭐⭐⭐ |
| **中国联通** | 充裕 | 最佳 | ⭐⭐⭐⭐⭐ |
| **中国移动** | 极小 | 最差 | ⭐ |

## 二、核心痛点：跨网互联

除了各运营商自身出口质量不同，**"跨网互联"才是导致同一节点速度天差地别的最大元凶**。

### 典型场景拆解

假设你买的机场，为节省成本只在国内购置了一台**单线电信服务器**作为入口：

- **电信用户** → 数据走电信内网，一路畅通，测速 500M✅
- **联通用户** → 数据需从联通跨越到电信网络，再接入入口服务器
- **移动用户** → 数据需从移动跨越到电信网络，才能连上入口服务器❌

### 问题根源

国内运营商之间的"跨网结算"费用极高，且互相限速（即"互联互通差"）。你的数据在从移动跨向电信的这座"独木桥"上，就被掐掉了大半带宽。

> **还没出国，网速就已经死在自家门口了。**

## 三、南北地域差异：物理距离不可逾越

除了运营商差异，**你的物理位置**也会带来巨大的测速差距。

### 国内出口骨干网分布

中国的国际出口骨干网主要集中于三座城市：

| 出口城市 | 主要覆盖方向 |
| :--- | :--- |
| **广州** | 香港、新加坡 |
| **上海** | 日本、美国 |
| **北京** | 北美、欧洲（部分） |

### 地域差距的实例

> **广东用户** 连香港节点：物理距离极近，延迟可能只有 10-20ms，快到飞起 🚀
>
> **哈尔滨用户** 连同一香港节点：数据先要跨越大半个中国到达广州，再从广州出海至香港。仅国内段数千公里的路程，就额外增加了几十毫秒延迟与大量带宽损耗 🐢

这也解释了为什么**测速图只能代表发图人所在网络环境下的表现**——"甲之蜜糖，乙之砒霜"在科学上网圈是绝对的真理。

## 四、普通用户的破局之法

既然运营商壁垒如此深厚，除了换宽带，我们还有什么选择？**当然有**。

在挑选机场时，只需关注以下两点：

### ✅ 首选"BGP 多线入口"机场

BGP 就像一个万能适配器。如果机场使用 BGP 入口，它会**自动分配与你同运营商的线路**进行对接，从入口就彻底消除跨网损耗，让电信、联通、移动三网用户都能体验极速。

> 关于 BGP 入口与专线的区别，可参考：[机场线路类型完全指南](/proxy/line-type-guide.html)

### ❌ 彻底放弃劣质直连节点

如果你的宽带是**移动**或**长城宽带**（二级运营商），请果断放弃价格低廉的直连节点，转而购买带有 **"BGP 中转"** 或 **"专线"** 的套餐。这是改善网速的最有效路径。

### 实用排查步骤

1. **确认自己的运营商**：打开路由器管理页或拨打运营商客服
2. **查看机场入口类型**：是否注明"BGP 多线""三网优化"
3. **测试时段选择晚高峰**（20:00–22:00），这才是最真实的使用场景
4. **对比多个地区节点**：同一机场，广东用户优先试香港节点；北方用户优先试日本/美国节点

## 五、总结

| 问题根源 | 说明 | 解决方案 |
| :--- | :--- | :--- |
| 运营商出口差异 | 联通最优，电信次之，移动最差 | 有条件可换联通宽带 |
| 跨网互联损耗 | 机场单线入口导致非对应运营商速度暴跌 | 选择 BGP 多线入口机场 |
| 地域物理距离 | 距出口城市越远延迟越高 | 选择距离最近的节点地区 |
| 直连节点质量 | 普通直连受高峰拥挤影响大 | 选择 BGP 中转或专线套餐 |
